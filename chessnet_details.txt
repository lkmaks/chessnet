	Основные таблицы (есть в обоих моделях)

>> Article - статья на сайте
author_username: автор статьи (пользователь, опубликовавший ее на сайте)
article_body: тело статьи (текст)
date_published: дата публикации

>> Comment - комментарий на сайте
// Комментарии оставляются только к статьям
article_id: статья, к которой оставлен комментарий
author_username: автор комментария
post_time: время, когда был опубликован комментарий
comment_body: тело (текст) комментария

>> OnlineUser - пользователь сайта, создаваемый при регистрации
username: имя пользователя
email: email
real_player_id: реальный игрок из базы данных, потдтвердивший, что владеет этим аккаунтом (аналогично "галочке" ВКонтакте)
password: пароль
decency_rating: рейтинг порядочности (отражает то, насколько дружелюбные/полезные пользователь пишет сообщения/комментарии)
chess_[blitz/rapid/standard]_rating: рейтинг данного пользователя в онлайн-партиях с соответствющим контролем времени
task_rating: рейтинг в решении задачы
[wins/draws/losses]_as_[white/black]: количество соотв. результатов за соотв. цвет

>> RealPlayer - реальный игрок
// Предполагается, что в этой таблице будут храниться игроки, сыгравшие хотя бы в одном 
// реальном (рейтинговом) турнире, и добавленные в последствии одним из админов
// Статистика ведется по тем же рейтинговым партиям.
name: Имя (человеческое) игрока
surname: Фамилия
[world/country]_chess_[blitz/rapid/standard]_rating: рейтинг (в международный / внутри страны) данного игрока с соответствющим контролем времени)
[wins/draws/losses]_as_[white/black]: количество соотв. результатов за соотв. цвет

>> Tournament - реальный турнир
name: название
date_from: дата начала
date_to: дата окончания
rule: правила (система турнира, временной контроль)
country: страна проведения
tournament_info: дополнительная информация для участников/зрителей в произвольной форме

>> Task - шахматная задача на сайте
author_username: имя пользователя, составившего эту задачу
online_game_from_id: онлайн-партия, из которой появилась эта задача (NULL если такой нет)
real_game_from_id: реальная партия, из которой появилась эта задача (NULL если такой нет)
task_body: код задачи (в нем содержится начальная позиция и дерево взаимодействия с решающим для интерактивности)
difficulty: сложность в некотором числовом эквиваленте
goal: что нужно сделать в задаче (выиграть материальное преимущество, поставить мат, зафорсить ничью, etc)
task_type: тип задачи (класс, к которому ее можно отнести, например "бесконечный шах" означает, что в основе решения лежит соотв. тактический прием)

>> OnlineGame - онлайн-партия
finish_time: когда закончилась партия
duration: сколько длилась партия
time_control: временной контроль (тип и конкретные значения)

>> RealGame - реальная партия
tournament_id: турнир, в котором была сыграна партия (у любой партии есть турнир; если это матч, создается фиктивный турнир)
game_date: дата, когда играли
time_control: временной контроль (тип и конкретные значения)
result: 0 - победа белых, 1 - ничья, 2 - победа черных

	Вспомогательные таблицы (есть только в логической модели)

>> UserInOnlineGame - объект, связывающий пользователя с партией, которую он играл (играет)
online_game_id: сама игра
username: сам пользователь
color: цвет, которым он играет (играл)
time_left: сколько у него на данный момент (или в конце партии, если она закончилась) осталось времени
result: результат партии для данного пользователя (2 - победа, 0 - поражение, 1 - ничья)

>> PlayerInTournament - объект, связывающий реального игрока с турниром, в котором он играл (играет)
tournament_id: сам турнир
real_player_id: сам игрок

>> PlayerInRealGame - объект, связывающий реального игрока с партией, которую он играл (играет)
real_player_id: сам игрок
real_game_id: сама игра
color:       \
time_left:    } аналогично UserInOnlineGame
